- 事前に定義されたSpecファイルに基づいて実装を開始せよ
  - ファイルパスが与えられた場合はそのSpecファイル、あるいはそのディレクトリ内にあるmarkdownファイルをspecとみなす
  - 何も与えられなかった場合、branch名のslash以降がslugになっている
    - 例 branch `feat/hybrid-vectorizer` なら `hybrid-vectorizer` がslug
    - `.cckiro/specs/{slug}/` 以下にmarkdownファイルがあるのでそれをspecファイルとみなせ
- Specファイルには進行状況を表現するためのチェックボックスがある
  - 完了した作業や編集が終わったファイルのチェックボックスにチェックを入れながら進めよ
  - チェックは最後にまとめてやるのではなく、都度実施せよ
- チェックボックスを埋めると同時にSpecの内容を見直し、実装すべき内容や方向性を確認せよ
- 実装中、追加の指示などあった場合はspecファイルもなるべく更新せよ
- 実装に支障をきたす問題が発生しない限り、最後まで完了させよ。途中で止まるな
